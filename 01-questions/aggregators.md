# Questions — merchant-side aggregators / PSPs

For each aggregator (pawaPay, DPO, MOKO, Onafriq, and any others found), answer:

- Does it operate in the **DRC**, and which operators does it cover there?
- **Collection** support? **Payout / disbursement** to any wallet or number?
- **Cross-network bridging** — can a payer on one network pay a payee on another through it?
- Does it offer **balance / wallet custody** (relevant if we hold a user balance via a partner)?
- **Pricing:** per-transaction fee per leg, plus the aggregator's margin; minimums.
- Settlement timing; currencies; reconciliation / transaction-status APIs.
- Developer access, docs quality, sandbox.
- Onboarding terms; does it provide **KYC-as-a-service**?
- Track record / scale (note when figures are **self-reported**).

Record answers in `02-findings/aggregators/<aggregator>.md`. Log sources in `04-sources/sources.md`.
