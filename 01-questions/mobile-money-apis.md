# Questions — mobile money operator APIs

For each operator (M-Pesa, Orange Money, Airtel Money, Afrimoney), answer:

- Is there a public developer API / portal? Where? How good are the docs?
- Is the API available for the **DRC specifically** (not just other markets)?
- Does it support **collection** (debit from a user's wallet)?
- Does it support **payout / disbursement** to any number on its network?
- What is the user **approval flow** (USSD/STK push, PIN, in-app)? Can it be smoother?
- **Fees** per transaction (collection and payout separately).
- Settlement timing; currencies supported (CDF / USD).
- Sandbox / test environment? Authentication model?
- Rate limits; transaction limits.
- Onboarding requirements for production access (who qualifies, KYC, contracts).
- Reliability / known issues.

Record answers in `02-findings/operators/<operator>.md`. Log sources in `04-sources/sources.md`.
